# プロジェクト指示文

## 基本ルール

### 1. 実装前の確認プロセス

- 実装に入る前に、理解した仕様を箇条書きで整理して提示すること。
- 不明点や曖昧な点があれば、推測で進めず必ず質問すること。
- 仕様の確認が取れてから実装を開始すること。

### 2. 影響範囲の事前調査

- 機能の追加・変更を行う際は、実装前に影響を受ける既存の関数・シート・トリガーを洗い出し、リストとして提示すること。
- 横展開が必要な箇所がある場合は、対応漏れがないよう明示すること。
- 影響範囲の確認が取れてから実装を開始すること。
- **通知メッセージは `buildNightMessage()`（夜の通知）と `buildMorningMessage()`（朝の通知）の2種類ある。** メッセージ内容に関する変更は、特に指定がない限り両方への影響を確認すること。

### 3. 段階的な進め方

- 大きな機能は一度に実装せず、以下のような段階に分けて進めること。
  1. データ構造・設計方針の提示 → 確認
  2. 主要ロジックの実装 → 確認
  3. スプレッドシートシートとの連携 → 確認
  4. 結合・仕上げ → 確認
- 各段階でユーザーの確認を取ってから次に進むこと。

### 4. その他

- 問題報告時は、まず原因を説明し、修正不要な場合はその旨を先に伝える。修正が必要な場合のみ対応方針を提示する。すぐに修正に飛びつかない。

## GAS実装上の注意点

### スプレッドシートへのアクセス

- シート名（`Config` / `Waste` / `Clothing` / `Log`）は定数化されておらず文字列直書きのため、シート名変更時は全箇所を確認すること。
- `getDataRange().getValues()` は全データを取得するため、大量行になる場合はパフォーマンスに注意すること。
- **GASコードとスプレッドシート（`data/ReadyGo.xlsx`）はセットで管理する。** コードの変更によりシート構造・値の形式が変わる場合は、必ずxlsxも同時に更新すること。
- `data/` 配下は実運用データを含むため `.gitignore` で公開リポジトリから除外している。ローカル環境にのみ存在する想定。
- **xlsxへの追記・変更時は、既存セルの書式（フォント・配置・罫線等）に合わせること。** 新規行は周囲のセルと書式を統一する。

### トリガー管理

- `setupTriggers()` は既存トリガーを**全削除**してから再作成する。意図しないトリガー削除に注意すること。
- `setDynamicMorningTrigger()` はGASの時間トリガーに±15分の誤差があるため、厳密な時刻制御には向かない点を考慮すること。
- トリガー変更を伴う実装の場合は、変更内容を事前に明示すること。

### 外部API呼び出し

- `UrlFetchApp.fetch()` は同期処理だが、GASの実行時間制限（6分）に注意すること。
- LINE APIのトークン・ユーザーIDはスクリプトプロパティから取得する。コード中にハードコードしないこと。

### 気象庁API（天気取得）の注意点

- 気温データは `timeSeries[2].areas[0].temps`（単一配列）で返ってくる。`tempsMin`/`tempsMax` は存在しない。
- `T00:xx+09:00` のエントリが最低気温、`T09:xx+09:00` のエントリが最高気温。
- GASの `new Date(t).getHours()` はUTCで解釈されるため誤った時刻になる。必ず `t.substring(11, 13)` で文字列から直接時間を取り出すこと。

## 技術スタック

- Google Apps Script (GAS)
- LINE Messaging API（プッシュ通知・リプライ・Webhook受信）
- Open-Meteo API（天気予報、認証不要）
- Google Calendar API（GAS組み込み `CalendarApp`）
- Google Spreadsheet（GAS組み込み `SpreadsheetApp`）

## プロジェクト構成

```
ReadyGo/
├── ReadyGo_Interactive.gs   ← メインスクリプト（全ロジック）
├── ReadyGo_仕様書.md        ← 仕様書
├── CLAUDE.md                ← 本ファイル
├── .gitignore               ← Git公開除外設定
└── data/                    ← ローカル運用データ（gitignore対象、公開しない）
    └── ReadyGo.xlsx         ← スプレッドシートのデータ定義（ローカル参照用）
```

### スプレッドシートのシート構成

| シート名 | 役割 |
|---|---|
| `Config` | 準備時間・移動時間・雨追加時間・弁当追加時間・傘閾値 |
| `Waste` | 曜日ごとのゴミの種類 |
| `Clothing` | 気温帯ごとの服装アドバイス |
| `Log` | 送信ログ（日時・種別・メッセージ） |

## API構成

実際のトークン・IDはすべてGASのスクリプトプロパティで管理する。コードにハードコードしないこと。

| 機能 | エンドポイント | 認証 |
|---|---|---|
| LINEプッシュ通知 | `https://api.line.me/v2/bot/message/push` | Bearer Token |
| LINEリプライ | `https://api.line.me/v2/bot/message/reply` | Bearer Token |
| 天気予報取得 | `https://api.open-meteo.com/v1/forecast` | 不要 |
| カレンダー取得 | GAS組み込み `CalendarApp` | OAuth（自動） |
| スプレッドシート | GAS組み込み `SpreadsheetApp` | OAuth（自動） |

### スクリプトプロパティ一覧

| キー名 | 説明 |
|---|---|
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Botのチャネルアクセストークン |
| `LINE_USER_IDS` | 通知先LINEユーザーID（複数の場合はカンマ区切り） |
| `SPREADSHEET_ID` | GoogleスプレッドシートのID |
| `JMA_AREA_CODE` | 気象庁の地域コード（省略時: `130000` 東京） |
