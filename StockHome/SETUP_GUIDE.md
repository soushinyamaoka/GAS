## 1. セットアップ手順

### 1.1 Google スプレッドシートを作成する

1. Google ドライブで「新規」→「Google スプレッドシート」を選択
2. ファイル名を **StockHome_Data** などに変更する
3. ブラウザの URL からスプレッドシート ID を控える
   - URL 例: `https://docs.google.com/spreadsheets/d/【ここがID】/edit`
   - `d/` と `/edit` の間の文字列がスプレッドシート ID

### 1.2 GAS プロジェクトを作成する

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を選択
2. GAS エディタが開く
3. プロジェクト名を **StockHome** に変更する

### 1.3 ソースコードを貼り付ける

`src/` フォルダ内のファイルを GAS エディタに貼り付ける。

#### .gs ファイル（サーバーサイド）

GAS エディタでファイルを追加する手順:
1. 左のファイル一覧の「＋」ボタン →「スクリプト」を選択
2. ファイル名を入力（`.gs` 拡張子は自動付与されるので不要）
3. コードを貼り付ける

以下のファイルを順に追加する:

| 順番 | ファイル名 | 役割 |
|------|-----------|------|
| 1 | Config | 定数・設定値 |
| 2 | Utils | 共通ユーティリティ |
| 3 | Validation | バリデーション |
| 4 | SheetRepository | シートアクセス共通化 |
| 5 | SheetInitializer | シート初期化 |
| 6 | UserService | ユーザー管理 |
| 7 | ItemService | 消耗品管理 |
| 8 | PurchaseService | 購入履歴管理 |
| 9 | ItemRuntimeStateService | 運用状態管理 |
| 10 | StockService | 在庫計算 |
| 11 | StockCorrectionService | 在庫補正 |
| 12 | NotificationService | 通知判定 |
| 13 | SnoozeService | スヌーズ管理 |
| 14 | ReadyGoBotService | ReadyGo Bot 連携（Inbox 投入） |
| 15 | GmailImportService | Gmail 取込 |
| 16 | AmazonMailParser | Amazon メール解析 |
| 17 | MatsukiyoMailParser | マツキヨメール解析 |
| 18 | TriggerService | トリガー管理 |
| 19 | BatchController | バッチ処理 |
| 20 | WebController | 画面向け関数 |
| 21 | Main | doGet エントリーポイント |

> **注意**: GAS エディタでは最初から存在する `コード.gs` を `Main` にリネームして使ってもよい。

#### .html ファイル（画面）

1. 左のファイル一覧の「＋」ボタン →「HTML」を選択
2. ファイル名を入力（`.html` 拡張子は自動付与されるので不要）
3. HTML コードを貼り付ける

| ファイル名 | 画面 |
|-----------|------|
| HtmlIndex | ホーム |
| HtmlItems | 消耗品一覧 |
| HtmlItemForm | 消耗品登録/編集 |
| HtmlPurchaseForm | 購入登録 |
| HtmlStocks | 在庫予測一覧 |
| HtmlImportCandidates | Gmail 取込候補 |
| HtmlGmailSettings | Gmail 設定 |
| HtmlNotifications | 通知履歴 |
| HtmlStockCorrection | 在庫補正 |

#### appsscript.json

1. GAS エディタの左メニュー「プロジェクトの設定」（歯車アイコン）を開く
2. 「「appsscript.json」マニフェスト ファイルをエディタで表示する」にチェックを入れる
3. エディタに戻ると `appsscript.json` が表示される
4. `src/appsscript.json` の内容で上書きする

### 1.4 Script Properties を設定する

1. GAS エディタのメニューから **「実行する関数を選択」** で `setupScriptProperties` を選ぶ
2. **「実行」** ボタンを押す
3. 初回は権限の承認を求められるので許可する
4. 実行ログを確認する
5. 左メニュー「プロジェクトの設定」→「スクリプト プロパティ」を開く
6. `SPREADSHEET_ID` の値を手順 1.1 で控えたスプレッドシート ID に書き換える

### 1.5 シートを初期化する

1. **「実行する関数を選択」** で `initializeAllSheets` を選ぶ
2. **「実行」** ボタンを押す
3. スプレッドシートを開いて、以下の 9 シートが作成されていることを確認する:
   - users
   - items
   - purchase_log
   - stock_snapshot
   - item_runtime_state
   - notification_log
   - import_order_candidates
   - stock_correction_log
   - app_config

## 2. デプロイ手順

### 2.1 Web アプリとしてデプロイする

1. GAS エディタの右上 **「デプロイ」→「新しいデプロイ」** を選択
2. 左の歯車アイコンから **「ウェブアプリ」** を選択
3. 以下のように設定する:

| 設定項目 | 値 |
|---------|-----|
| 説明 | StockHome v1.0 |
| 次のユーザーとして実行 | **ウェブアプリにアクセスしているユーザー** |
| アクセスできるユーザー | 家族の Google アカウントに応じて選択 ※ |

> ※ 家族全員が同じ Google Workspace ドメインの場合: 「組織内の全員」
> ※ 個人 Gmail の場合: 「全員」（Google アカウントが必要）を選択
> ※ Google アカウントを持つユーザーのみアクセスできる

4. **「デプロイ」** を押す
5. デプロイ URL が表示されるのでコピーする

### 2.2 WEBAPP_BASE_URL と READYGO_SPREADSHEET_ID を設定する

1. GAS エディタ →「プロジェクトの設定」→「スクリプト プロパティ」
2. `WEBAPP_BASE_URL` にデプロイ URL を設定する
3. `READYGO_SPREADSHEET_ID` に ReadyGo Bot 側のスプレッドシート ID を設定する
   - 値は ReadyGo 管理者から共有してもらう
   - 未設定でも StockHome 自体は動作するが、夜の LINE 通知に在庫アラートが流れない
4. ReadyGo Bot のスプレッドシートに、StockHome を実行する Google アカウント（= 日次バッチを所有するユーザー）を **編集者** として追加してもらう
   - Inbox シートへの行追加権限が必要

### 2.3 スプレッドシートの共有設定

1. スプレッドシートを開く
2. 右上の「共有」ボタンを押す
3. 家族全員の Google アカウントを **「編集者」** として追加する

> **重要**: 家族全員にスプレッドシートの編集権限が必要。
> Web アプリは「アクセスしているユーザー」として実行するため、
> 各ユーザーがスプレッドシートに直接書き込む。

### 2.4 日次バッチ trigger を作成する

1. GAS エディタで `createDailyBatchTrigger` を選択して実行する
2. 毎晩 20:00〜21:00 に `runDailyBatch` が自動実行されるようになる
3. ReadyGo Bot が 21:00 に LINE 通知を送るタイミングの直前に在庫計算とアラート投入が完了する設計

> 日次バッチは「誰のトリガーか」に依存しない共通処理。
> 管理者（最初にセットアップする人）が 1 回だけ作れば OK。

### 2.5 コードを更新した場合

1. GAS エディタでコードを修正する
2. **「デプロイ」→「デプロイを管理」** を選択
3. 既存のデプロイの右上の鉛筆アイコンを押す
4. バージョンを **「新バージョン」** に変更する
5. **「デプロイ」** を押す

> URL は変わらないが、新しいバージョンが反映される。

## 3. 家族ユーザーの初回利用手順

家族の各メンバーが以下を行う。

### 手順

1. 管理者から共有されたデプロイ URL をスマホのブラウザで開く
2. Google アカウントでのログインを求められるのでログインする
3. 初回は「このアプリは確認されていません」と表示されることがある
   - 「詳細」→「（安全でないページ）に移動」で進む
   - これは個人開発アプリでは正常な動作
4. 権限承認のダイアログが出るので「許可」を押す
5. ホーム画面が表示される
6. 「ユーザーが未登録です」と表示された場合:
   - 下部ナビの右端や、ホーム画面のリンクから **Gmail 設定** 画面に移動する
   - ユーザー名を入力して「ユーザーを保存」を押す
7. ホームに戻ると自分の名前が表示される

### ブックマーク推奨

- スマホのブラウザで「ホーム画面に追加」すると、アプリのように使える
- URL はデプロイ URL そのまま（`?page=home` は省略可）

## 4. Gmail 自動取込の有効化手順

各ユーザーが **自分で** 以下を行う。

### 手順

1. Web アプリの **Gmail 設定** 画面を開く
2. 「Gmail 自動取込を有効化」ボタンを押す
3. Gmail の読み取り権限の承認を求められるので許可する
4. 「Gmail 自動取込を有効化しました（6時間ごとに実行されます）」と表示される
5. 必要に応じて検索クエリを調整し「検索条件を保存」を押す
6. 「今すぐ実行」ボタンで初回の取込を手動実行できる

### 検索クエリの初期値

| vendor | 初期クエリ |
|--------|-----------|
| Amazon | `from:(auto-confirm@amazon.co.jp OR shipment-tracking@amazon.co.jp)` |
| マツキヨ | `from:(*@matsukiyo.co.jp)` |

### クエリのカスタマイズ例

```
# Amazon で件名も含めて絞りたい場合
from:(auto-confirm@amazon.co.jp OR shipment-tracking@amazon.co.jp) subject:(注文 OR 発送)

# マツキヨで特定アドレスに限定したい場合
from:online-shop@matsukiyo.co.jp
```

### 動作の仕組み

- ボタン押下時に、そのユーザー自身の installable trigger が作成される
- trigger は 6 時間ごとに `runMyGmailImport` を呼ぶ
- `runMyGmailImport` 実行時、**そのユーザーの Gmail だけ** が検索される
- ソースコードは全員共通だが、trigger の実行ユーザーが違うため読む Gmail が異なる

### 無効化するには

1. Gmail 設定画面で「Gmail 自動取込を無効化」を押す
2. trigger が削除される

## 5. テスト手順

### 5.1 基本動作テスト（最初にやること）

#### Step 1: ユーザー登録

1. Web アプリを開く
2. Gmail 設定画面でユーザー名を入力して保存
3. スプレッドシートの `users` シートに行が追加されたことを確認

#### Step 2: 消耗品登録

1. 「消耗品を追加」画面を開く
2. 以下のテストデータを入力して保存:

| 項目 | 値 |
|------|-----|
| 品名 | トイレットペーパー |
| カテゴリ | 日用品 |
| 単位 | パック |
| 標準購入数 | 1 |
| 1単位あたり消費日数 | 30 |
| 通知何日前 | 7 |
| 安全日数 | 3 |
| 在庫数しきい値通知数 | 1 |

3. スプレッドシートの `items` シートと `item_runtime_state` シートに行が追加されたことを確認

#### Step 3: 購入登録

1. 「購入を記録」画面を開く
2. Step 2 で追加した品目を選択
3. 購入日: 今日、購入数: 1 で登録
4. `purchase_log` シートに行が追加されたことを確認

#### Step 4: 在庫予測確認

1. 「在庫」画面を開く
2. Step 2 の品目が表示され、推定残日数が約 30 日であることを確認

#### Step 5: 在庫補正

1. 「在庫を補正」画面を開く
2. Step 2 の品目を選択
3. 現在の推定残数が表示されることを確認
4. 実際の残数に 0.5 を入力、理由「実数を数えた」で補正
5. `stock_correction_log` に履歴が追加されたことを確認
6. `item_runtime_state` の `manual_override_qty` が 0.5 になったことを確認

#### Step 6: 消耗品編集・削除

1. 消耗品一覧で「編集」を押す
2. 品名を変更して保存
3. 消耗品一覧で「削除」を押す
4. 確認ダイアログで OK を押す
5. `items` シートで `is_active = FALSE` になったことを確認

### 5.2 通知テスト

1. 消耗品を登録し、`days_per_unit = 1`（1日で1個消費）、`notify_target_type = all` に設定する
2. 購入日を 1 日前にして購入数 1 で登録する
3. GAS エディタで `runDailyBatch` を手動実行する
4. 以下を確認:
   - `stock_snapshot` に在庫計算結果が保存される
   - `notification_log` に通知レコード（`target_user_id = broadcast`）が作成される
   - ReadyGo 側スプレッドシートの `Inbox` シートに 1 行追加され、`source = StockHome` の本文に集約メッセージが入っている

### 5.3 スヌーズテスト

1. 在庫予測一覧で「3日スヌーズ」を押す
2. `item_runtime_state` の `snooze_until` に 3 日後の日時が入ることを確認
3. `runDailyBatch` を手動実行しても、スヌーズ中の品目は ReadyGo Inbox に出力されないことを確認
4. 「スヌーズ解除」を押し、再度 `runDailyBatch` を実行して通知が生成されることを確認

### 5.4 Gmail 取込テスト

1. Gmail 設定画面で「今すぐ実行」を押す
2. GAS の実行ログで取込結果を確認
3. Amazon やマツキヨの注文メールがあれば `import_order_candidates` に候補が作成される
4. 取込候補画面で品目を選択して「確定」を押す
5. `purchase_log` に反映されることを確認

> Gmail にテスト用のメールがない場合は、自分宛に以下の件名でメールを送ると
> パーサーの動作を確認できる（本文にそれらしい内容を書く）:
> - 件名: `Amazon.co.jp ご注文の確認`
> - 件名: `商品を発送しました`

## 6. ReadyGo Bot 連携

StockHome は LINE 配信を自前で行わず、既存の **ReadyGo Bot**（家族向け生活自動化Bot）の 21:00 LINE 通知に在庫アラートを追記してもらう仕組みになっている。

### 6.1 責務分担

```
┌─────────────────────────────┐      ┌─────────────────────────────┐
│ StockHome                    │      │ ReadyGo Bot                  │
│                             │      │                             │
│ ・在庫計算                   │      │ ・Inbox シートを読み取り       │
│ ・通知判定                   │      │ ・夜 21:00 の LINE 通知に追記  │
│ ・通知文面生成（集約）        │      │ ・配信完了で processed=TRUE   │
│ ・Inbox に1行追加 ───────────│──→   │                             │
│ ・notification_log に記録    │      │                             │
└─────────────────────────────┘      └─────────────────────────────┘
         ↕                                       ↕
              ReadyGo 側スプレッドシート（Inbox シート）
```

### 6.2 StockHome 側の動作

- 日次バッチ（毎晩 20:00〜21:00）の終盤で `NotificationService.processAllNotifications` が動作
- アラート対象品目を抽出し、**1つのメッセージにまとめて** ReadyGo Inbox に1行投入
- 対象品目の条件:
  - `is_active = TRUE`
  - `notification_enabled = TRUE`
  - `alert_needed = TRUE`（残日数または残数しきい値超過）
  - スヌーズ中でない
  - `notify_target_type = all`（ReadyGo はブロードキャストのため）
- 重複防止は現状無効（同じ品目が連日アラートになっても毎日通知される）
- ReadyGo Inbox 投入が失敗した場合は `notification_log` への記録もスキップされ、バッチ自体は継続

### 6.3 Inbox シートの列構成（ReadyGo 側仕様）

| 列 | 名前 | StockHome が書き込む値 |
|----|------|---------------------|
| A | posted_at | 投入時刻（`new Date()`） |
| B | source | `StockHome` 固定 |
| C | body | 集約メッセージ本文 |
| D | processed | `false` 固定（ReadyGo 側で配信後 `true` に更新） |

### 6.4 投入メッセージのフォーマット

```
📦 在庫アラート (3件)

・ティッシュ：残3日 / 約2個
・牛乳：残2日 / 約1.0L
・米：しきい値以下 (残0.5袋)

→ 在庫予測画面で確認
https://script.google.com/macros/s/.../exec?page=stocks
```

- 件数を冒頭に表示
- 1品目1行: `品目名：残N日` / `しきい値以下 (残N単位)` / `残N日 / 残N単位`
- 末尾に WEBAPP_BASE_URL から在庫予測画面への直リンク

### 6.5 必要な設定

1. **スクリプトプロパティ `READYGO_SPREADSHEET_ID`** に ReadyGo 側スプレッドシート ID を設定
2. **ReadyGo 側スプレッドシートの共有設定** で、StockHome の日次バッチを所有する Google アカウントに編集者権限を付与
3. これらが揃わない場合、`appendToInbox` はログに記録するだけでバッチは継続する（夜の LINE 通知から在庫アラートが欠落するだけ）

### 6.6 失敗時の挙動

- `READYGO_SPREADSHEET_ID` 未設定 → Logger.log のみ、`notification_log` 未記録
- 編集権限なし / シート不在 / その他例外 → Logger.log のみ、`notification_log` 未記録
- ReadyGo 側の Log シートも合わせて確認すると配信状況が分かる

## 7. 注意点と運用上のコツ

### 7.1 GAS の実行時間制限

- GAS は **1 回の実行が最大 6 分** まで
- 消耗品が数十品目程度なら問題ない
- Gmail 取込は MAX_THREADS = 30 に制限しているので通常は問題ない
- 品目数が 100 を超えるような場合は、日次バッチの処理を分割する検討が必要

### 7.2 Gmail 取込のタイミング

- trigger は 6 時間ごとに実行される
- 「今すぐ実行」を使えば手動で即時実行もできる
- 初回は過去 30 日分を検索するため、少し時間がかかることがある
- 2 回目以降は「前回取込日時 - 3 日」以降のみ検索するので軽い

### 7.3 メールパーサーの精度

- Amazon / マツキヨのメール形式は **変更される可能性がある**
- パーサーが商品名を取れない場合は件名がフォールバックで使われる
- パース失敗時もエラー候補として保存されるので、手動で確認できる
- 正規表現パターンは `AmazonMailParser.gs` / `MatsukiyoMailParser.gs` の先頭に集約してあるので、調整しやすい

### 7.4 在庫計算の精度

- 在庫計算は「最新購入からの線形消費」という単純モデル
- 実際の消費ペースとずれることは前提
- **在庫補正機能を積極的に使う** のが運用のコツ
- 補正すると、補正日を起点に消費計算が再開される
- 新しい購入を登録すると、古い補正値は自動クリアされる

### 7.5 通知の重複送信について

- 現状、StockHome 側では重複防止を行わない（同じアラートが連日 ReadyGo Inbox に投入される）
- ReadyGo 側では1度配信されると `processed=true` になり再配信されないため、過配信は当日内に閉じる
- 翌日のバッチで同じ品目が再びアラート対象なら、改めて Inbox 投入される
- 将来再導入する場合は `NotificationService.hasRecentNotification` を呼び出す形に戻す（関数は残置済）

### 7.6 スプレッドシートが大きくなったら

- `purchase_log` や `notification_log` は追記型なので時間とともに増える
- 数千行程度なら問題ないが、パフォーマンスが気になったら:
  - 古い通知ログの行を手動で削除する
  - `stock_correction_log` の古い行を削除する
- `items`, `users`, `stock_snapshot`, `item_runtime_state` は品目数分なので増えにくい

### 7.7 複数人の同時操作

- 書き込み系の操作には `LockService` を使っているので、データ破損は防がれる
- ロック待ちタイムアウト（10 秒）を超えると「他の処理が実行中です」エラーが出る
- 家族数人が同時に使う程度なら、ほぼ問題にならない

### 7.8 未確認アプリの警告

- GAS の Web アプリは Google に審査されていないため「このアプリは確認されていません」と表示される
- 「詳細」→「（安全でないページ）に移動」で先に進める
- これは個人開発 GAS アプリでは正常な動作であり、セキュリティ上の問題ではない
- 家族全員に初回だけこの手順を案内しておく

### 7.9 権限の再承認が必要になるケース

以下の場合、ユーザーに再度権限承認を求められることがある:
- `appsscript.json` の `oauthScopes` を変更した場合
- ユーザーが Google アカウントの権限設定でアプリの権限を取り消した場合
- Gmail 取込 trigger が削除された場合（設定画面から再有効化する）

### 7.10 バックアップ

- スプレッドシートは Google ドライブ上にあるので、自動的にバージョン管理される
- 心配であれば、定期的にスプレッドシートを「コピーを作成」しておくとよい

### 7.11 日常の運用サイクル

```
毎日（自動）:
  夜 20:00 → StockHome 日次バッチ（在庫計算 + ReadyGo Inbox 投入）
  夜 21:00 → ReadyGo Bot が LINE 通知に在庫アラートを追記して配信
  6h ごと → Gmail 取込（各ユーザーの trigger）

家族の日常操作:
  ・店舗で買い物したら → 購入登録
  ・在庫を確認したら → 在庫補正
  ・LINE 通知が来たら → 買いに行く or スヌーズ
  ・Gmail 取込候補が溜まったら → 確認画面で確定/無視
```

## 8. 既存環境からの移行手順

旧バージョン（line_outbox + 既存 LINE Bot 経由）から ReadyGo 連携版へ移行する場合は以下を実施する。

### 8.1 コードを最新版にデプロイする

通常通り `push.bat` → `deploy.bat` を実行。

### 8.2 スクリプトプロパティ `READYGO_SPREADSHEET_ID` を追加する

セクション 2.2 参照。ReadyGo 管理者からスプレッドシート ID を共有してもらう。

### 8.3 ReadyGo スプレッドシートに編集権限を付与してもらう

ReadyGo 側スプレッドシートに、StockHome の日次バッチを所有する Google アカウントを編集者として追加してもらう。

### 8.4 日次バッチトリガーを再作成する

朝7時から夜20時に移動するため、既存トリガーを作り直す。

1. GAS エディタで `deleteDailyBatchTrigger` を実行（旧トリガーを削除）
2. 続けて `createDailyBatchTrigger` を実行（新トリガーを 20時で作成）
3. GAS エディタの「トリガー」画面（左メニュー時計アイコン）で 20時設定になっていることを確認

### 8.5 `users` シートに email を設定する

ホーム画面のアラートをユーザー別に絞り込むため、`users` シートの `email` 列に各ユーザーの Google アカウントメールアドレスを記入する。未設定でも StockHome は動作するが、`notify_target_type` フィルタは効かず全件表示になる。

**自動補完の仕組み（2026年5月実装）**:
- ユーザー登録 / 更新時に email 列が空欄なら、`Session.getActiveUser().getEmail()` で自動取得した値が保存される
- 既存ユーザー（email が空欄のままの人）は、Web アプリの **Gmail 設定画面で「ユーザーを保存」を1度押すだけ** で email が補完される（UserProperties に user_id がキャッシュされている前提）
- UserProperties が失われている（新ブラウザ等）場合は、`users` シートで該当行の email を手動入力するか、管理者がスプレッドシートで直接編集する

### 8.6 `line_outbox` シートタブを削除（任意）

旧フローのデータが残っていても害はないが、不要なら手動でシートタブを削除して構わない。

1. スプレッドシートを開く
2. 下部の `line_outbox` シートタブを右クリック → 削除
3. 確認ダイアログで OK

### 8.7 動作確認

セクション 5.2 のテスト手順で ReadyGo Inbox に行が追加されることを確認する。
