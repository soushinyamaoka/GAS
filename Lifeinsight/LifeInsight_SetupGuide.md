# ライフ・インサイト セットアップガイド

## 全体の流れ

1. Googleフォームを作成する
2. スプレッドシートにフォーム回答を連携する
3. スプレッドシートの列見出しを準備する
4. GASにコードを貼り付ける
5. PropertiesServiceを設定する（setupProperties を実行）
6. トリガーを設定する（2つ）

---

## 手順1：Googleフォームの作成

1. [Google Forms](https://forms.google.com) を開き、「空白のフォーム」を作成
2. フォームのタイトルを「ライフ・インサイト」に設定
3. 以下の質問を順番に作成する

| 順番 | 質問タイトル | 質問の種類 | 選択肢 / 設定 |
|------|-------------|-----------|--------------|
| 1 | 気分 | ラジオボタン | 1, 2, 3, 4, 5 |
| 2 | 体調 | ラジオボタン | 1, 2, 3, 4, 5 |
| 3 | 睡眠の質 | ラジオボタン | 1, 2, 3 |
| 4 | メモ | 記述式（短文） | ※必須にしない |
| 5 | チェック項目 | チェックボックス（複数選択） | 薬, カフェイン, 酒 |

4. 各スケール項目の補足（任意）
   - 気分: 1=最悪 〜 5=最高
   - 体調: 1=最悪 〜 5=最高
   - 睡眠の質: 1=悪い, 2=普通, 3=良い

---

## 手順2：フォーム回答をスプレッドシートに連携

1. フォームの編集画面で上部の「回答」タブをクリック
2. 「スプレッドシートにリンク」（緑のアイコン）をクリック
3. 「新しいスプレッドシートを作成」を選択 → 「作成」

これで「フォームの回答 1」シートが自動生成されます。

---

## 手順3：スプレッドシートの列見出しを追加

フォーム回答シートの **G1〜N1** に以下の見出しを入力してください。
A〜F列はフォーム連携で自動設定済みのため触りません。

| セル | 見出し |
|------|--------|
| G1 | 瞬間気圧 (hPa) |
| H1 | 気圧トレンド (hPa) |
| I1 | 24h変動幅 (hPa) |
| J1 | 気温 (℃) |
| K1 | 寒暖差 (℃) |
| L1 | 湿度 (%) |
| M1 | 月齢 |
| N1 | 休日フラグ |

---

## 手順4：GASにコードを貼り付ける

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く
2. エディタに最初から書いてある `function myFunction() {}` を全て削除する
3. `LifeInsight.gs` の中身を全てコピーして貼り付ける
4. `Ctrl + S`（またはフロッピーアイコン）で保存する

---

## 手順5：PropertiesServiceを設定する

### 5-1. スプレッドシートIDの確認

スプレッドシートのURLからIDを取得します。

```
https://docs.google.com/spreadsheets/d/【ここがスプレッドシートID】/edit
```

例: `1aBcDeFgHiJkLmNoPqRsTuVwXyZ` のような文字列です。

### 5-2. 緯度・経度の確認

自宅の緯度・経度を調べます。
Google Mapsで自宅付近を右クリック → 表示される座標をメモしてください。

例（東京駅）: 緯度 `35.6812` / 経度 `139.7671`

### 5-3. setupProperties を編集・実行

GASエディタで `setupProperties` 関数の以下の3行を自分の値に書き換えます。

```javascript
props.setProperty('SPREADSHEET_ID', '実際のスプレッドシートIDをここに');
props.setProperty('LAT', '35.6812');    // 自宅の緯度
props.setProperty('LON', '139.7671');   // 自宅の経度
```

書き換えたら：
1. 上部のドロップダウンで `setupProperties` を選択
2. 「▶ 実行」ボタンをクリック
3. 初回実行時に「承認が必要です」と表示されたら：
   - 「権限を確認」をクリック
   - Googleアカウントを選択
   - 「詳細」→「〇〇（安全ではないページ）に移動」をクリック
   - 「許可」をクリック
4. 実行ログに「PropertiesService の設定が完了しました」と表示されればOK

---

## 手順6：トリガーを設定する（2つ）

GASエディタの左メニューから「時計マーク（トリガー）」をクリックし、以下の2つを設定します。

### トリガー1: フォーム送信時

1. 「トリガーを追加」をクリック
2. 以下のように設定する

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `onFormSubmit` |
| イベントのソース | スプレッドシートから |
| イベントの種類 | フォーム送信時 |

3. 「保存」をクリック

### トリガー2: 日次更新（毎日0:05）

1. 「トリガーを追加」をクリック
2. 以下のように設定する

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `updateDailySummary` |
| イベントのソース | 時間主導型 |
| 時間ベースのトリガーのタイプ | 日付ベースのタイマー |
| 時刻 | 午前0時〜1時 |

3. 「保存」をクリック

※ GASの日付ベースタイマーは正確な分指定ができないため、「午前0時〜1時」の間に実行されます。

---

## 動作確認

1. Googleフォームを開いてテスト送信する
2. スプレッドシートの「フォームの回答 1」シートを開く
3. 送信した行のG〜N列にデータが自動入力されていればOK

### データが入っていない場合の確認手順

1. GASエディタで「実行数」（左メニュー）を開く
2. `onFormSubmit` の実行結果を確認する
3. エラーが出ていれば「Log」シートにも記録されているか確認する

---

## シート構成まとめ

| シート名 | 用途 |
|----------|------|
| フォームの回答 1 | フォーム回答 + GAS自動追記の気象データ |
| 気象ログ | フォーム入力がなかった日の気象データ（自動作成） |
| Log | エラーログ（自動作成） |
